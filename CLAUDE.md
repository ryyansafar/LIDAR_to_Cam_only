# LiDAR-to-Camera Height Prediction for Pothole Detection

## Project Overview

Train a model on paired LiDAR + camera data so that **at inference time only a camera image is needed** to produce a dense height map of the road surface. Potholes are detected as localized depressions in the predicted height map.

## Dataset: RSRD-dense

- **Train**: `RSRD-dense/train/` — 27 timestamped sequences, each with `pcd/`, `left/`, `depth/` subdirs
- **Test**: `RSRD-dense/test/` — 300 frames, flat structure (no sequences)
- **Road segmentation**: `road_segmentation/` — 63 PCD files with `x,y,z,label` (label=1 is road)
- Resolution: 1920×1080, files named by timestamp e.g. `20230317074850.000`

## Calibration

Real calibration values from [RSRD_dev_toolkit](https://github.com/ztsrxh/RSRD_dev_toolkit/tree/main/calibration_files).
Stored in `config.yaml` under `calibration:` with per-date keys.

| Date prefix | fx = fy | cx | cy |
|---|---|---|---|
| 20230317, 20230321 | 2024.51 | 1033.38 | 498.57 |
| 20230406, 20230408, 20230409 | 2022.60 | 1037.42 | 500.82 |

**To use real calibration**: edit the `calibration:` section in `config.yaml`. Keys are 8-digit date strings matching the sequence folder name prefix (e.g. `20230408`).

## Quick Start

```bash
# 1. Install dependencies
pip install -r requirements.txt

# 2. Verify calibration (visual check — critical before training)
python src/data/precompute_heights.py --config config.yaml --verify --n_verify 5
# → Saves overlays to results/verify/. Check that LiDAR dots land ON the road surface.

# 3. Precompute height maps (run once, ~4-6 hours on CPU, ~1h on good CPU)
python src/data/precompute_heights.py --config config.yaml --split all --workers 4

# 4. Compute height statistics (updates config.yaml automatically)
python src/data/precompute_heights.py --config config.yaml --stats

# 5. Train
python src/train.py --config config.yaml

# 6. Evaluate on test set
python src/evaluate.py --config config.yaml --checkpoint checkpoints/best.pt --save_vis

# 7. Inference on new image
python src/inference.py --image path/to/image.jpg --checkpoint checkpoints/best.pt
```

## Projection Math

Standard pinhole model with LiDAR→camera extrinsics:
```
P_cam = R @ P_lidar + T
depth = P_cam[2]                    # camera-frame Z
u = fx * P_cam[0] / depth + cx
v = fy * P_cam[1] / depth + cy
height_map[v, u] = P_lidar[2]      # store LiDAR Z (elevation)
```

Intrinsics are scaled to the output resolution (640×360) before projection.

## Pothole Detection

Post-processing the predicted height map:
1. Gaussian blur → local road plane estimate (sigma=25 by default)
2. `residual = local_plane - height_map` (positive = depression)
3. Threshold at 5cm below plane → binary mask
4. Morphological open/close → connected components → bounding boxes

Tune `detection.pothole_depth_threshold` in `config.yaml` for sensitivity.

## Troubleshooting

**LiDAR points not on road surface** (in `--verify` output):
→ The calibration values need updating. The real calibration PKLs are at
  https://github.com/ztsrxh/RSRD_dev_toolkit/tree/main/calibration_files

**Validation MAE plateaus above 0.15m**:
→ This indicates calibration or data quality issues. Check `--verify` output first.
→ With real calibration, target MAE < 0.1m within 20 epochs.

**CUDA out of memory**:
→ Reduce `training.batch_size` in `config.yaml` (try 2 or 1).

**Resume training**:
```bash
python src/train.py --config config.yaml --resume checkpoints/last.pt
```
